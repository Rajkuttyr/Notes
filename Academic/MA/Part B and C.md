# Linear Multimedia

## Definition

Linear multimedia is a type of multimedia in which the content is presented in a **fixed, sequential order**, and the user **cannot control or interact** with the flow of content.

The presentation moves from beginning to end automatically.

## Explanation

In linear multimedia, the system controls the presentation. The user acts as a passive viewer and has no control over navigation, sequence, or content flow.

The information is presented in a predetermined order designed by the creator.

## Characteristics of Linear Multimedia

• Fixed sequence  
• No user interaction  
• User has no control  
• Passive user participation  
• Continuous playback  
• Predefined content flow  
• Simple structure

## Examples of Linear Multimedia

1. Movies shown in theatres
    
2. Television programs
    
3. Recorded video lectures
    
4. Music playback
    
5. Animated films
    
6. Digital advertisements
    
7. Presentation videos

When watching a movie, scenes play in a fixed order from beginning to end. The viewer cannot change the sequence or storyline.

## Definition

Graphic file formats are used to store **still images, drawings, and pictures** in digital form.

• Color information

---

## Types of Graphic File Formats

### 1. BMP (Bitmap File Format)

**Definition:**  
BMP is a raster graphic format used to store images without compression.

**Features:**  
• Developed by Microsoft  
• High quality image  
• Large file size  
• No compression

**Example:**  
photo.bmp

---

### 2. JPEG (Joint Photographic Experts Group)

**Definition:**  
JPEG is a compressed image format used for storing photographic images.

**Features:**  
• Lossy compression  
• Small file size  
• Good quality  
• Most commonly used format

**Example:**  
photo.jpg

---

### 3. PNG (Portable Network Graphics)

**Definition:**  
PNG is a compressed image format that supports lossless compression.

**Features:**  
• Lossless compression  
• High quality  
• Supports transparency  
• Used in websites

**Example:**  
image.png

---

### 4. GIF (Graphics Interchange Format)

**Definition:**  
GIF is an image format that supports animation and limited colors.

**Features:**  
• Supports animation  
• Supports transparency  
• Limited to 256 colors

**Example:**  
animation.gif

**Definition:**  
TIFF is a high-quality image format used in professional applications.

**Features:**  
• High quality  
• Large file size

**Example:**  
image.tiff

# Definition

Object linking and embedding are techniques used to **integrate multimedia objects such as images, videos, and documents into applications**.

Distributed object management refers to managing objects located on different computers in a network.

---

# 1. Object Linking

## Definition

Object linking is the process of connecting an object to its source file.

The object is not stored in the document, only the link is stored.

## Features

• Saves storage space  
• Automatically updates when source changes  
• Requires source file availability

## Example

Linking an image in a Word document.

If the original image changes, the linked image also updates.

---

# Advantages

• Less storage required  
• Automatic update

# Disadvantages

• Source file must be available

---

# 2. Object Embedding

## Definition

Object embedding is the process of inserting an object directly into a document.

The object becomes part of the document.

## Features

• Stored inside the document  
• Independent of source file  
• Larger file size

## Example

Embedding an image into a Word document.

Even if original image is deleted, embedded image remains.

---

# Advantages

• No dependency on source  
• Easy to share

# Disadvantages

• Larger file size

---

# Difference Between Linking and Embedding

|Linking|Embedding|
|---|---|
|Stores link|Stores object|
|Smaller file size|Larger file size|
|Depends on source|Independent|
|Auto updates|Does not update|

---

# 3. Managing Distributed Objects

## Definition

Distributed object management refers to managing multimedia objects stored on different computers connected through a network.

---

## Features

• Objects stored on different systems  
• Accessed through network  
• Supports distributed multimedia applications

---

## Examples

• Cloud storage  
• Web applications  
• Online multimedia systems

YouTube videos stored on different servers.

---

## Advantages

• Efficient resource sharing  
• Easy access from anywhere

---

# Conclusion

Object linking and embedding help integrate multimedia objects into documents. Linking stores reference, embedding stores actual object. Distributed object management allows multimedia objects to be stored and accessed across networks efficiently.

## Introduction

In multimedia, different types of files are used such as images, audio, video, text, and animation. These files are created using different software and devices. To make sure that these files can be opened and used on different systems without any problem, certain rules and standards are followed.

These rules are called file standards. They help in maintaining quality and compatibility in multimedia projects.

---

# Definition (2 Marks)

File standards in multimedia are predefined rules or specifications that define how multimedia files (audio, video, images, text, animation) should be stored, compressed, and shared so that they can work properly on different devices and software.

In simple words, file standards ensure that multimedia files can be used everywhere without errors.

---

# Importance of File Standards (5 Marks)

File standards are very important in multimedia for the following reasons:

---

## 1. Ensures Compatibility Across Devices

Multimedia files are used on computers, mobile phones, tablets, smart TVs, and many other devices. If there were no standards, a file created in one device may not open in another.

Example:  
An MP4 video file can be played on almost all devices because it follows a standard format.

This makes sharing easy and smooth.

---

## 2. Maintains Quality of Content

Standards define how audio and video should be compressed and stored. This helps in maintaining good sound and picture quality.

Example:  
JPEG standard ensures acceptable image quality even after compression.  
MP3 standard maintains clear audio with smaller file size.

Thus, standards help balance quality and storage size.

---

## 3. Reduces File Size Efficiently

Multimedia files can be very large. Standards define proper compression techniques so that files become smaller without losing too much quality.

Example:  
MPEG standard reduces video size for streaming.  
PNG uses lossless compression for images.

This is important for internet usage and storage.

---

## 4. Supports Interoperability

Interoperability means different systems working together smoothly. File standards allow files created in one software to be opened in another.

Example:  
A PDF file can be opened in many different PDF readers.  
An MP3 file can be played in any music player.

This makes multimedia production and sharing easier.

---

## 5. Saves Time and Cost

When common standards are followed, developers and companies do not need to create new formats for every project. This reduces development time and cost.

It also avoids compatibility issues that may cause extra expenses.

---

# Role of File Standards in Ensuring Compatibility and Quality (3 Marks)

---

## 1. Provides Uniform Structure

File standards define a fixed structure for storing data. Because of this uniform structure, devices and software understand how to read the file correctly.

This ensures compatibility across platforms.

---

## 2. Defines Compression and Encoding Methods

Standards specify how audio, video, and images should be compressed and encoded. Proper encoding ensures that quality is maintained while reducing file size.

For example:

- MP3 defines audio compression method.
    
- MPEG defines video compression method.
    

This ensures consistent quality.

---

## 3. Enables Global Sharing

Since file standards are accepted worldwide, multimedia content can be shared globally without conversion problems.

For example:  
A YouTube video in MP4 format can be viewed anywhere in the world.

Thus, standards play a key role in global communication.

---

# Conclusion

File standards are essential in multimedia because they ensure that files can be opened, shared, and used across different devices and platforms. They help maintain quality, reduce file size, and support compatibility.

Without file standards, multimedia files would face many problems like poor quality, large size, and incompatibility between devices.

Therefore, file standards play a major role in maintaining both compatibility and quality in multimedia systems.
