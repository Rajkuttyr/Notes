# 🧠 What is Flood Fill?

Flood fill means:

👉 Starting from one cell  
👉 Change its color  
👉 Also change all **connected cells with same original color**

Basically: “spread like water”.

---

## 🎨 Example

Input:

```
1 1 1
1 1 0
1 0 1
```

Start at (1,1) and change to color `2`

Output:

```
2 2 2
2 2 0
2 0 1
```

Only connected `1`s are changed.

---

# 🔥 Approach

Flood fill is just:

👉 DFS (Depth First Search)  
OR  
👉 BFS (Breadth First Search)

It’s just graph traversal on a grid.

---

# 🚀 Java DFS Solution (Interview Ready)

```java
class Solution {
    public int[][] floodFill(int[][] image, int sr, int sc, int color) {
        
        int original = image[sr][sc];
        
        if(original == color) return image; // avoid infinite loop
        
        dfs(image, sr, sc, original, color);
        return image;
    }
    
    private void dfs(int[][] image, int r, int c, int original, int color) {
        
        if(r < 0 || c < 0 || 
           r >= image.length || c >= image[0].length ||
           image[r][c] != original) {
            return;
        }
        
        image[r][c] = color;
        
        dfs(image, r+1, c, original, color);
        dfs(image, r-1, c, original, color);
        dfs(image, r, c+1, original, color);
        dfs(image, r, c-1, original, color);
    }
}
```

---

# 🧠 Why We Check `original == color`

If user asks to change `1 → 1`

Without that check, recursion runs forever.

Very important edge case.

---

# ⏱ Time Complexity

If grid is `m x n`

Worst case:

```
O(m * n)
```

We may visit all cells once.
