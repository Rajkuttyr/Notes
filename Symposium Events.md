# 1. Blind Coding Challenge

![Image](https://images.shiksha.com/mediadata/images/articles/1745408915phpcJTdRb.jpeg)

![Image](https://cs.utdallas.edu/files/2015/12/HS-Battle-Brains-Prog-2015-Cont-Students-11.webp)

![Image](https://miro.medium.com/0%2AgJWTCYMQcEvfaiz5)

![Image](https://www.masaischool.com/blog/content/images/2022/06/Competitive-Programming.png)

## Objective

Test pure logic without relying on compiler output.

## Setup

- Individual or team of 2
    
- Provide paper or laptop (compiler disabled initially)
    

## Rounds

### Round 1: Easy (Elimination)

- Write code for basic problems  
    Example: Reverse a string
    
- Duration: 20 mins
    
- Top 50% qualify
    

### Round 2: Medium

- Slightly complex problems  
    Example: Find second largest element
    
- Duration: 30 mins
    
- Top 5 or 10 qualify
    

### Round 3: Final Round (Blind Execution)

- Students write code without running it
    
- Compiler allowed only once at end
    

## Winner Selection

Criteria:

- Correct output
    
- Code accuracy
    
- Least errors
    
- Fastest correct solution

Test ability to use AI effectively.

## Setup

- Team of 2
    
- Internet + AI tools allowed
    

## Rounds

### Round 1: Prompt Optimization

Task: Generate best explanation for a topic
